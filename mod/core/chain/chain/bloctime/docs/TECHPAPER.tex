\section{Introduction}

BlocTime represents a modular blockchain protocol designed for decentralized asset management, staking mechanisms, and marketplace operations. The protocol consists of six core contracts working in harmony to provide a complete DeFi ecosystem.

\section{Core Contracts}

\subsection{BlocTime Token \& Staking Contract}

\subsubsection{Mathematical Model}

The BlocTime contract implements a time-weighted staking mechanism where users earn tokens based on stake duration:

\begin{equation}
B_{earned} = A_{staked} \times M(t)
\end{equation}

where:
\begin{itemize}
\item $B_{earned}$ = BlocTime tokens earned
\item $A_{staked}$ = Amount of native tokens staked
\item $M(t)$ = Multiplier function based on lock duration $t$
\end{itemize}

\subsubsection{Multiplier Calculation}

The multiplier uses linear interpolation between configured points:

\begin{equation}
M(t) = M_0 + \frac{(M_1 - M_0) \times (t - t_0)}{t_1 - t_0}
\end{equation}

for $t_0 \leq t \leq t_1$, where $(t_0, M_0)$ and $(t_1, M_1)$ are consecutive points in the multiplier curve.

\subsubsection{Key Features}
\begin{itemize}
\item ERC20 token with staking functionality
\item Configurable lock periods up to $maxLockBlocks$
\item Multiple stake positions per user
\item Non-decreasing multiplier curve enforcement
\item Reentrancy protection
\end{itemize}

\subsection{Market Contract}

\subsubsection{Pricing Model}

The Market contract implements a stable token system with whitelisted payment tokens:

\begin{equation}
P_{required} = \frac{S_{amount} \times 10^{d_{token}}}{P_{token}}
\end{equation}

where:
\begin{itemize}
\item $P_{required}$ = Payment amount required
\item $S_{amount}$ = Stable token amount desired
\item $d_{token}$ = Token decimals
\item $P_{token}$ = Token price (8 decimals USD)
\end{itemize}

\subsubsection{Operations}
\begin{itemize}
\item \textbf{Credit}: Mint stable tokens by paying with whitelisted tokens
\item \textbf{Debit}: Burn stable tokens from user balance
\item Token whitelist management
\item Manual price oracle integration
\end{itemize}

\subsection{Treasury Contract}

\subsubsection{Distribution Model}

The treasury implements proportional distribution based on governance token ownership:

\begin{equation}
C_{holder} = \frac{D_{total} \times (10000 - P_{owner})}{10000} \times \frac{B_{holder}}{S_{total}}
\end{equation}

where:
\begin{itemize}
\item $C_{holder}$ = Claimable amount for holder
\item $D_{total}$ = Total distributable amount
\item $P_{owner}$ = Owner percentage (basis points)
\item $B_{holder}$ = Holder's governance token balance
\item $S_{total}$ = Total governance token supply
\end{itemize}

\subsubsection{Multi-Token Support}
\begin{itemize}
\item Support for multiple treasury tokens
\item Proportional claiming across all tokens
\item Owner percentage allocation
\item Cumulative claim tracking
\end{itemize}

\subsection{Permissions Contract}

\subsubsection{Hierarchical Model}

The Perms contract implements a parent-child key relationship:

\begin{equation}
K_{parent} \rightarrow [K_{child1}, K_{child2}, ..., K_{childn}]
\end{equation}

with constraints:
\begin{itemize}
\item $n \leq maxChildKeys$
\item $|K_{child}| \leq maxKeySize$
\end{itemize}

\subsubsection{Access Control}
\begin{itemize}
\item Parent key ownership model
\item Configurable limits (gas optimization)
\item Ownership transfer capability
\item Event-driven architecture
\end{itemize}

\section{Integration Patterns}

\subsection{Contract Interaction Flow}

\begin{enumerate}
\item User stakes native tokens in BlocTime contract
\item BlocTime tokens minted based on duration multiplier
\item BlocTime tokens used as governance tokens in Treasury
\item Treasury distributes multiple token types proportionally
\item Market enables stable token minting via whitelisted tokens
\item Registry tracks asset metadata on-chain
\item Perms manages hierarchical access control
\end{enumerate}
